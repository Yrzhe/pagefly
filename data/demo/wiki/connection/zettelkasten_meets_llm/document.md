# Zettelkasten Meets LLM Wiki

Karpathy's LLM Wiki proposal and Luhmann's Zettelkasten are separated by a
century but converge on the same core claim: **the unit of knowledge is the
atomic idea, not the source document**.

## What they share

| Zettelkasten | LLM Wiki |
|--------------|----------|
| One idea per note | One concept per wiki page |
| Explicit links between notes | Wiki cross-references and a graph |
| Human summarizes the source | LLM summarizes the source |
| Organized by idea, not by topic | Same |

## Where they differ

The Zettelkasten assumes a disciplined human doing all the work. The LLM Wiki
assumes the human has no discipline and needs the machine to do it.

Both systems answer the same question — "what should my knowledge base look
like?" — but from opposite ends of the labor/automation spectrum.

## A synthesis

PageFly is a Zettelkasten where the LLM is the librarian. The atomic-idea rule
is enforced by the compiler agent (every concept page is atomic by
construction). The linking is automatic. The human's only job is to curate
sources — the part that's genuinely hard to automate because it requires taste.

# Changelog

## [Unreleased]

### Changed
- **Desktop â€” API token storage moved from Keychain to a 0600 file** (`~/Library/Application Support/PageFly/api_token`). Keychain ACLs are bound to the binary's cdhash, and ad-hoc rebuilds change cdhash on every build, so the prior design popped a "PageFly Capture wants to use confidential information stored in your keychain" dialog after every reinstall. The new `TokenStore` is a single `Data.write(.atomic)` + `chmod 600`; trades Keychain's per-binary ACL for POSIX home-directory protection â€” the right tradeoff for a personal ad-hoc build, and the surface (`SettingsStore.apiToken` getter, `setApiToken(_:)`) is unchanged so callers don't notice. `probeToken()` falls back to a one-time Keychain read on first launch after upgrading; if it finds a legacy token it migrates the value into the file and deletes the Keychain entry, so subsequent rebuilds never touch Keychain again. A future Developer ID-signed release can swap back to Keychain (stable cdhash â†’ no prompts).

### Added
- **Desktop â€” Pause/Resume capture from the menu bar**. New Pause button in the popover (next to Record / Test / Dashboard) flips `CapturePipeline.isPausedByUser`, which short-circuits `sampleNow` without tearing down workspace observers or the sampler timer â€” resume is instant. Header subtitle changes to "Paused", body text explains that uploads of already-queued events continue in the background, and the menu bar icon tint turns `systemYellow` via a new `.paused` state in `MenuBarController`. Pause is intentionally ephemeral (no UserDefaults) so quitting + relaunching always restores capture â€” prevents the "I paused last Tuesday and forgot" failure mode.
- **Desktop â€” Deeper AX capture for Electron / browser apps**. Walk budgets bumped (`maxNodes` 400â†’1500, `maxDepth` 12â†’20, `maxTextChars` 1500â†’4000). Three Screenpipe-derived improvements on top of the budget changes: (1) **Chromium tree materialization** â€” `AXEnhancedUserInterface=true` + `AXManualAccessibility=true` set on the app element per pid (TTL 180s to avoid re-triggering the expensive tree rebuild each poll); without these flags Chromium-based apps (Cursor, Slack, Notion, Discord, VSCode, Obsidian) return a stripped AX tree. Refs Chromium codereview.chromium.org/6909013, electron/electron#7206, obsidianmd#3002. (2) **Decorative role skip** â€” whole subtrees rooted at `AXScrollBar/AXImage/AXSplitter/AXGrowArea/AXMenuBar/AXMenu/AXToolbar/AXRuler/AXBusyIndicator/AXProgressIndicator/AXLayoutItem` are pruned so node budget reaches the actual content pane in Electron chrome-heavy UIs. (3) **Wall-clock budget** â€” walk gets a 300ms `deadline` on top of the node/depth caps; a pathological slow app can no longer burn `maxNodes Ă— per-element timeout` seconds on the main thread. Per-element messaging timeout tightened 0.8s â†’ 0.3s since wall-clock now backs it. Text collection also reads `kAXSelectedTextAttribute` (many Electron editors only expose text via selection). Reverted the `kAXVisibleChildrenAttribute` fallback from the previous iteration â€” cost an extra IPC round-trip per element and Electron apps routinely return `[]` for visible-children, net-negative; Screenpipe also walks `kAXChildrenAttribute` directly. Browser URL now tries `AXDocument` on the window first before descending into `AXWebArea` â€” Safari/Chrome/Edge expose it without needing the web tree walk.

### Fixed
- **Web â€” Pending capture rows had no way to see full content on small days.** The inline `Recent rows` list in `PendingDayCard` truncated text with `line-clamp-2` and was not clickable, and the `View all N events` modal button was gated on `event_count > samples.length`. That meant days with `event_count â‰¤ samples` (12 by default from `/api/activity/pending`) rendered a dead-end truncated list with no escape hatch. Fix: (1) new `InlineSampleRow` â€” same click-to-expand pattern as `DetailRow`, shows full `text_excerpt` in a `<pre>` with `max-h-240px overflow-y-auto` plus the URL as an external link; (2) modal button always surfaces when `event_count > 0` ("View all N" when there are more events than samples, "Open in full view" otherwise) so the scrollable-modal path is never gated off.

### Fixed
- **Desktop â€” Dashboard events sometimes showed row 1 as newer than row 0** on rapid app switches. `fetchRecentEvents` used `ORDER BY COALESCE(ended_at, started_at) DESC` with no tie-break, so during the few-millisecond window where a just-closed row's final `ended_at` equals (or narrowly exceeds) a brand-new live row's `started_at`, non-deterministic row order would put the older row on top. Added `started_at DESC, created_at DESC` as tie-breakers.

### Added
- **Desktop â€” AXReader now harvests page text + URL**. Previous build only read the focused element's `kAXValueAttribute` / `kAXSelectedTextAttribute`, which was nil for browser windows, chat windows, and most container UIs â€” so `text_excerpt` was empty for nearly every captured event. AXReader now bounded-walks the focused window (depth â‰¤ 12, â‰¤ 400 nodes, 0.4s per-element AX messaging timeout to bound main-thread cost on huge browser DOMs) and collects `kAXValue` / `kAXTitle` / `kAXDescription` from descendants, plus the URL from the first `AXWebArea` it encounters. Persisted text still prefers the focused element when present (active typing context wins) and falls back to harvested page text. Walk skips subtrees whose role is in `PrivacyFilter.blockedAXRoles`.
- **Desktop â€” Dashboard pagination**. Events list now pages through `local_events` instead of capping at the newest 250: 100 rows per page with prev/next buttons + `Page X of Y` indicator next to the section header. New `LocalDB.eventCount()` drives the counter; `fetchRecentEvents` gained an `offset` parameter. Page index is clamped on every refresh so deleting the only row on page N falls back to page N-1 instead of rendering an empty list.
- **Desktop â€” Dashboard window**. New "Dashboardâ€¦" button in the popover opens a standalone window (resizable, 760Ă—600 default). Top fixed-height (240px) section lists every recording most-recent first with an inline play/stop button (`AVAudioPlayer` via `AudioPreview` singleton), duration, status, age, and a transcript preview when available. Bottom scrollable section shows the 250 newest captured events with app/window/URL/text excerpt. Both lists expose a 2-step "trash â†’ Confirm/Cancel" delete only on rows the server hasn't accepted yet (`local_audio.status IN ('pending_upload','failed')` and `local_events.status = 'pending'`); deleting an audio row also removes its m4a from disk and stops playback if it was the active preview. Auto-refreshes every 4s while the window is open. Backed by new `LocalDB.fetchRecentEvents`, `deletePendingEvent`, `deleteLocalAudio`, `fetchAudio` helpers â€” each delete re-checks status under the write lock so a row that flipped to `uploading` between render and click is left alone with a user-visible "already uploading" message.
- **Desktop (M8) â€” menu bar UX polish + dashboard + audio retention**. Status icon is now a 6-state machine (noConfig / noPermission / offline / checking / armed / recording) with baked-in tints (`labelColor` for neutral, system semantic colors for active states) so it reads correctly on both light and dark menu bars; popover gains an Accessibility-permission banner with a single "Grantâ€¦" button that focuses the right System Settings pane and force-refreshes a shared `AXTrustMonitor`. New audio retention slider in Preferences (0â€“50, default 5) â€” `AudioUploader.vacuumExpiredFiles()` enforces the keep-N cap on every flush in addition to the existing 7-day time cleanup; transcripts are kept on the server regardless. New `desktop-capture/scripts/package-local.sh` produces an ad-hoc-signed DMG for personal use without an Apple Developer account.

### Fixed
- **Desktop â€” uploads stalling at 405**. Default server URL was `https://pagefly.ink`, which now serves the static landing-page SPA and rejects POST. Changed default to `https://api.pagefly.ink`; added a one-shot migration in `SettingsStore.init` so existing installs on the legacy default get rewritten silently.
- **Desktop â€” `prepareToRecord` failed under hardened runtime**. `package-local.sh` was re-signing with `codesign --force --deep --sign -` (no `--entitlements`), which silently stripped the microphone / audio-input entitlements and left the hardened runtime blocking `AVAudioRecorder`. Re-sign now passes `-o runtime --entitlements Resources/PageflyCapture.entitlements` and the script aborts if the microphone entitlement is missing post-sign.
- **Desktop â€” main-thread hang at launch on rebuilt ad-hoc binaries**. `SettingsStore` lazy init was calling `Keychain.load` on the main thread; under ad-hoc signing, `securityd` would block on an access-control prompt that LSUIElement apps can never surface, freezing the menu bar before the popover existed. Init no longer touches Keychain; `probeToken()` runs the read on a detached `Task` and publishes `hasToken` when it returns.

- **Desktop (M7) â€” launch-at-login + in-app updater + release script**. `SMAppService.mainApp` wired behind a Preferences â†’ General "Launch at login" toggle; surfaces "requires approval" when macOS asks the user to flip the switch in System Settings â†’ Login Items. New `UpdateChecker` polls the GitHub Releases API every 6h, compares against `CFBundleShortVersionString`, and exposes a "Check for Updates" / "Download" button in the About tab (opens the release HTML page in the browser â€” no auto-install). Hardened the URL path: repo slug is regex-validated (tamper-resistant against `defaults write`) and `html_url` host must live under `github.com`. New `desktop-capture/scripts/release.sh` wraps `xcodegen` + `xcodebuild archive` + `codesign` + `xcrun notarytool submit --wait` + `stapler staple`; export plist is built via `plutil -insert` so Developer ID identities with `&`/`<`/`>` don't corrupt the XML.
- **Desktop auto-capture â€” server side**. New `activity_events` + `audio_recordings` tables; helpers are idempotent on client-issued `local_uuid`. New endpoints under `/api/activity/*`: multipart audio upload with async Whisper transcription (atomic O_EXCL write, path-traversal-safe uuid validation), transcription status polling, batch event ingest (max 1000 events, FK-safe â€” unknown `audio_id` references are dropped to NULL with a warning), and event retrieval by date range. Canonical append-only log at `data/activity/YYYY-MM-DD.jsonl`, raw audio at `data/activity/audio/<uuid>.<ext>`. New `activity_log` agent summarizes a day's events + transcripts into a "Work log YYYY-MM-DD" wiki article; scheduled daily at 08:00 (registered in `/api/schedules` system list).
- **Scheduled task execution history** â€” every scheduled task run (system and user) is now recorded to the `task_runs` table with status (running/success/failed), started/finished timestamps, output, error, duration, and source (cron vs. user-triggered).
- **Run Now button** on user schedule cards â€” trigger a task immediately via `POST /api/schedules/{task_id}/run-now`; runs asynchronously and appears in history when complete.
- **Expandable run history** on the Schedules panel â€” click a schedule to view its recent runs; click a run to open a modal with full output / error and metadata.
- **`ingest` task type** implemented â€” dispatches to the query agent (same brain as the Telegram bot), so scheduled ingest/custom tasks can use all MCP tools (read/write knowledge, search, ingest URLs, compile wiki, etc.).
- New API endpoints: `GET /api/schedules/{task_id}/runs`, `GET /api/schedule-runs/recent`, `GET /api/schedule-runs/{run_id}`, `POST /api/schedules/{task_id}/run-now`.
- Telegram notifications now include a `Run #{id}` reference and truncate long output (3500 char cap) with a pointer to the Schedules tab for the full output.

### Changed

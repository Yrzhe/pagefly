# Changelog

All notable changes to PageFly are documented in this file. The format is based on [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Changed
- **Desktop — API token storage moved from Keychain to a 0600 file** (`~/Library/Application Support/PageFly/api_token`). Keychain ACLs are bound to the binary's cdhash, and ad-hoc rebuilds change cdhash on every build, so the prior design popped a "PageFly Capture wants to use confidential information stored in your keychain" dialog after every reinstall. The new `TokenStore` is a single `Data.write(.atomic)` + `chmod 600`; trades Keychain's per-binary ACL for POSIX home-directory protection — the right tradeoff for a personal ad-hoc build, and the surface (`SettingsStore.apiToken` getter, `setApiToken(_:)`) is unchanged so callers don't notice. `probeToken()` falls back to a one-time Keychain read on first launch after upgrading; if it finds a legacy token it migrates the value into the file and deletes the Keychain entry, so subsequent rebuilds never touch Keychain again. A future Developer ID-signed release can swap back to Keychain (stable cdhash → no prompts).

### Added
- **Desktop — AXReader now harvests page text + URL**. Previous build only read the focused element's `kAXValueAttribute` / `kAXSelectedTextAttribute`, which was nil for browser windows, chat windows, and most container UIs — so `text_excerpt` was empty for nearly every captured event. AXReader now bounded-walks the focused window (depth ≤ 12, ≤ 400 nodes, 0.4s per-element AX messaging timeout to bound main-thread cost on huge browser DOMs) and collects `kAXValue` / `kAXTitle` / `kAXDescription` from descendants, plus the URL from the first `AXWebArea` it encounters. Persisted text still prefers the focused element when present (active typing context wins) and falls back to harvested page text. Walk skips subtrees whose role is in `PrivacyFilter.blockedAXRoles`.
- **Desktop — Dashboard pagination**. Events list now pages through `local_events` instead of capping at the newest 250: 100 rows per page with prev/next buttons + `Page X of Y` indicator next to the section header. New `LocalDB.eventCount()` drives the counter; `fetchRecentEvents` gained an `offset` parameter. Page index is clamped on every refresh so deleting the only row on page N falls back to page N-1 instead of rendering an empty list.
- **Desktop — Dashboard window**. New "Dashboard…" button in the popover opens a standalone window (resizable, 760×600 default). Top fixed-height (240px) section lists every recording most-recent first with an inline play/stop button (`AVAudioPlayer` via `AudioPreview` singleton), duration, status, age, and a transcript preview when available. Bottom scrollable section shows the 250 newest captured events with app/window/URL/text excerpt. Both lists expose a 2-step "trash → Confirm/Cancel" delete only on rows the server hasn't accepted yet (`local_audio.status IN ('pending_upload','failed')` and `local_events.status = 'pending'`); deleting an audio row also removes its m4a from disk and stops playback if it was the active preview. Auto-refreshes every 4s while the window is open. Backed by new `LocalDB.fetchRecentEvents`, `deletePendingEvent`, `deleteLocalAudio`, `fetchAudio` helpers — each delete re-checks status under the write lock so a row that flipped to `uploading` between render and click is left alone with a user-visible "already uploading" message.
- **Desktop (M8) — menu bar UX polish + dashboard + audio retention**. Status icon is now a 6-state machine (noConfig / noPermission / offline / checking / armed / recording) with baked-in tints (`labelColor` for neutral, system semantic colors for active states) so it reads correctly on both light and dark menu bars; popover gains an Accessibility-permission banner with a single "Grant…" button that focuses the right System Settings pane and force-refreshes a shared `AXTrustMonitor`. New popover **Dashboard** section: live counts of queued events + audio waiting to upload, plus the last 5 recordings with status (recording / waiting / transcribing / transcribed / failed) and age. Recording state shows a `TimelineView`-driven elapsed timer. New audio retention slider in Preferences (0–50, default 5) — `AudioUploader.vacuumExpiredFiles()` enforces the keep-N cap on every flush in addition to the existing 7-day time cleanup; transcripts are kept on the server regardless. App icon (`AppIcon.appiconset`) and menu bar template (`MenuBarIcon.imageset`) generated procedurally via `desktop-capture/scripts/generate-icons.py`. New `desktop-capture/scripts/package-local.sh` produces an ad-hoc-signed DMG for personal use without an Apple Developer account.

### Fixed
- **Desktop — uploads stalling at 405**. Default server URL was `https://pagefly.ink`, which now serves the static landing-page SPA and rejects POST. Changed default to `https://api.pagefly.ink`; added a one-shot migration in `SettingsStore.init` so existing installs on the legacy default get rewritten silently.
- **Desktop — `prepareToRecord` failed under hardened runtime**. `package-local.sh` was re-signing with `codesign --force --deep --sign -` (no `--entitlements`), which silently stripped the microphone / audio-input entitlements and left the hardened runtime blocking `AVAudioRecorder`. Re-sign now passes `-o runtime --entitlements Resources/PageflyCapture.entitlements` and the script aborts if the microphone entitlement is missing post-sign.
- **Desktop — main-thread hang at launch on rebuilt ad-hoc binaries**. `SettingsStore` lazy init was calling `Keychain.load` on the main thread; under ad-hoc signing, `securityd` would block on an access-control prompt that LSUIElement apps can never surface, freezing the menu bar before the popover existed. Init no longer touches Keychain; `probeToken()` runs the read on a detached `Task` and publishes `hasToken` when it returns.

- **Desktop (M7) — launch-at-login + in-app updater + release script**. `SMAppService.mainApp` wired behind a Preferences → General "Launch at login" toggle; surfaces "requires approval" when macOS asks the user to flip the switch in System Settings → Login Items. New `UpdateChecker` polls the GitHub Releases API every 6h, compares against `CFBundleShortVersionString`, and exposes a "Check for Updates" / "Download" button in the About tab (opens the release HTML page in the browser — no auto-install). Hardened the URL path: repo slug is regex-validated (tamper-resistant against `defaults write`) and `html_url` host must live under `github.com`. New `desktop-capture/scripts/release.sh` wraps `xcodegen` + `xcodebuild archive` + `codesign` + `xcrun notarytool submit --wait` + `stapler staple`; export plist is built via `plutil -insert` so Developer ID identities with `&`/`<`/`>` don't corrupt the XML.
- **Desktop auto-capture — server side**. New `activity_events` + `audio_recordings` tables; helpers are idempotent on client-issued `local_uuid`. New endpoints under `/api/activity/*`: multipart audio upload with async Whisper transcription (atomic O_EXCL write, path-traversal-safe uuid validation), transcription status polling, batch event ingest (max 1000 events, FK-safe — unknown `audio_id` references are dropped to NULL with a warning), and event retrieval by date range. Canonical append-only log at `data/activity/YYYY-MM-DD.jsonl`, raw audio at `data/activity/audio/<uuid>.<ext>`. New `activity_log` agent summarizes a day's events + transcripts into a "Work log YYYY-MM-DD" wiki article; scheduled daily at 08:00 (registered in `/api/schedules` system list).
- **Scheduled task execution history** — every scheduled task run (system and user) is now recorded to the `task_runs` table with status (running/success/failed), started/finished timestamps, output, error, duration, and source (cron vs. user-triggered).
- **Run Now button** on user schedule cards — trigger a task immediately via `POST /api/schedules/{task_id}/run-now`; runs asynchronously and appears in history when complete.
- **Expandable run history** on the Schedules panel — click a schedule to view its recent runs; click a run to open a modal with full output / error and metadata.
- **`ingest` task type** implemented — dispatches to the query agent (same brain as the Telegram bot), so scheduled ingest/custom tasks can use all MCP tools (read/write knowledge, search, ingest URLs, compile wiki, etc.).
- New API endpoints: `GET /api/schedules/{task_id}/runs`, `GET /api/schedule-runs/recent`, `GET /api/schedule-runs/{run_id}`, `POST /api/schedules/{task_id}/run-now`.
- Telegram notifications now include a `Run #{id}` reference and truncate long output (3500 char cap) with a pointer to the Schedules tab for the full output.

### Changed
- Unified `custom` and `ingest` scheduled task types — both delegate to `src/agents/query.py:ask()`; the stored `task_type` still drives filtering and colors in the UI.
- Scheduler wraps every task in `_run_with_recording` — task failures never crash the scheduler, DB write failures log warnings but don't fail the task, notify failures are swallowed.

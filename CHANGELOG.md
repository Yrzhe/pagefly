# Changelog

All notable changes to PageFly are documented in this file. The format is based on [Keep a Changelog](https://keepachangelog.com/).

## [Unreleased]

### Added
- **Desktop (M8) ‚Äî menu bar UX polish + dashboard + audio retention**. Status icon is now a 6-state machine (noConfig / noPermission / offline / checking / armed / recording) with baked-in tints (`labelColor` for neutral, system semantic colors for active states) so it reads correctly on both light and dark menu bars; popover gains an Accessibility-permission banner with a single "Grant‚Ä¶" button that focuses the right System Settings pane and force-refreshes a shared `AXTrustMonitor`. New popover **Dashboard** section: live counts of queued events + audio waiting to upload, plus the last 5 recordings with status (recording / waiting / transcribing / transcribed / failed) and age. Recording state shows a `TimelineView`-driven elapsed timer. New audio retention slider in Preferences (0‚Äì50, default 5) ‚Äî `AudioUploader.vacuumExpiredFiles()` enforces the keep-N cap on every flush in addition to the existing 7-day time cleanup; transcripts are kept on the server regardless. App icon (`AppIcon.appiconset`) and menu bar template (`MenuBarIcon.imageset`) generated procedurally via `desktop-capture/scripts/generate-icons.py`. New `desktop-capture/scripts/package-local.sh` produces an ad-hoc-signed DMG for personal use without an Apple Developer account.

### Fixed
- **Desktop ‚Äî uploads stalling at 405**. Default server URL was `https://pagefly.ink`, which now serves the static landing-page SPA and rejects POST. Changed default to `https://api.pagefly.ink`; added a one-shot migration in `SettingsStore.init` so existing installs on the legacy default get rewritten silently.
- **Desktop ‚Äî `prepareToRecord` failed under hardened runtime**. `package-local.sh` was re-signing with `codesign --force --deep --sign -` (no `--entitlements`), which silently stripped the microphone / audio-input entitlements and left the hardened runtime blocking `AVAudioRecorder`. Re-sign now passes `-o runtime --entitlements Resources/PageflyCapture.entitlements` and the script aborts if the microphone entitlement is missing post-sign.
- **Desktop ‚Äî main-thread hang at launch on rebuilt ad-hoc binaries**. `SettingsStore` lazy init was calling `Keychain.load` on the main thread; under ad-hoc signing, `securityd` would block on an access-control prompt that LSUIElement apps can never surface, freezing the menu bar before the popover existed. Init no longer touches Keychain; `probeToken()` runs the read on a detached `Task` and publishes `hasToken` when it returns.

- **Desktop (M7) ‚Äî launch-at-login + in-app updater + release script**. `SMAppService.mainApp` wired behind a Preferences ‚Üí General "Launch at login" toggle; surfaces "requires approval" when macOS asks the user to flip the switch in System Settings ‚Üí Login Items. New `UpdateChecker` polls the GitHub Releases API every 6h, compares against `CFBundleShortVersionString`, and exposes a "Check for Updates" / "Download" button in the About tab (opens the release HTML page in the browser ‚Äî no auto-install). Hardened the URL path: repo slug is regex-validated (tamper-resistant against `defaults write`) and `html_url` host must live under `github.com`. New `desktop-capture/scripts/release.sh` wraps `xcodegen` + `xcodebuild archive` + `codesign` + `xcrun notarytool submit --wait` + `stapler staple`; export plist is built via `plutil -insert` so Developer ID identities with `&`/`<`/`>` don't corrupt the XML.
- **Desktop auto-capture ‚Äî server side**. New `activity_events` + `audio_recordings` tables; helpers are idempotent on client-issued `local_uuid`. New endpoints under `/api/activity/*`: multipart audio upload with async Whisper transcription (atomic O_EXCL write, path-traversal-safe uuid validation), transcription status polling, batch event ingest (max 1000 events, FK-safe ‚Äî unknown `audio_id` references are dropped to NULL with a warning), and event retrieval by date range. Canonical append-only log at `data/activity/YYYY-MM-DD.jsonl`, raw audio at `data/activity/audio/<uuid>.<ext>`. New `activity_log` agent summarizes a day's events + transcripts into a "Work log YYYY-MM-DD" wiki article; scheduled daily at 08:00 (registered in `/api/schedules` system list).
- **Scheduled task execution history** ‚Äî every scheduled task run (system and user) is now recorded to the `task_runs` table with status (running/success/failed), started/finished timestamps, output, error, duration, and source (cron vs. user-triggered).
- **Run Now button** on user schedule cards ‚Äî trigger a task immediately via `POST /api/schedules/{task_id}/run-now`; runs asynchronously and appears in history when complete.
- **Expandable run history** on the Schedules panel ‚Äî click a schedule to view its recent runs; click a run to open a modal with full output / error and metadata.
- **`ingest` task type** implemented ‚Äî dispatches to the query agent (same brain as the Telegram bot), so scheduled ingest/custom tasks can use all MCP tools (read/write knowledge, search, ingest URLs, compile wiki, etc.).
- New API endpoints: `GET /api/schedules/{task_id}/runs`, `GET /api/schedule-runs/recent`, `GET /api/schedule-runs/{run_id}`, `POST /api/schedules/{task_id}/run-now`.
- Telegram notifications now include a `Run #{id}` reference and truncate long output (3500 char cap) with a pointer to the Schedules tab for the full output.

### Changed
- Unified `custom` and `ingest` scheduled task types ‚Äî both delegate to `src/agents/query.py:ask()`; the stored `task_type` still drives filtering and colors in the UI.
- Scheduler wraps every task in `_run_with_recording` ‚Äî task failures never crash the scheduler, DB write failures log warnings but don't fail the task, notify failures are swallowed.
